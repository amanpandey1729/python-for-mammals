Today you will combine lists, dictionaries, and functions to build a practical inventory workflow for servers, environments, owners, status, and capacity reporting.

What you will build

Day 18 is a project day. You will not learn a random new syntax item. You will combine the previous building blocks into one useful automation-style script.

Lists

Many servers

A list stores multiple server records so your script can loop through inventory data.

Dictionaries

One server record

A dictionary stores structured details like name, environment, OS, owner, status, CPU, memory, and disk.

Functions

Reusable actions

Functions create records, print lines, search inventory, filter records, and build summaries.

Search

Find a server

Search by server name and return either the matching dictionary or None when not found.

Filter

Group records

Filter production, UAT, active, inactive, or maintenance servers for quick operational reporting.

Capacity

Summarize totals

Add CPU, memory, and disk values across all inventory records to produce capacity summaries.
