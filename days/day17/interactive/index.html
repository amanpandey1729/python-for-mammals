What you will learn

Today builds directly on Day 16 functions. The big shift is this: functions should not always print. Often, they should return values that the main script can reuse.

return

Send value back

A function can calculate disk usage, ticket age, or health status and return the result to the main script.

Local Variables

Temporary function data

Variables created inside a function are local. They keep internal calculations safe and separate.

Global Variables

Outside function data

Global variables are created outside functions. Functions can read them, but parameters are usually cleaner.

Reusable Functions

Calculate, decide, report

Build small functions that calculate values, return statuses, and support clean report generation.

Automation Pattern

Input once, reuse often

Collect values once, pass them into functions, store returned values, and generate a final workflow report.

Cleaner Scripts

Less repeated logic

When calculation logic lives in functions, your scripts become easier to test, change, and reuse.
