Day 11: Store multiple values with Lists.

Today you will learn how to create lists, access values using indexes, and update values. Lists are the first step toward handling multiple servers, files, alerts, checks, and report items in automation scripts.

Start Interactive Practice Jump to Mini Challenge

One variable stores one value. A list stores many related values. That is why lists are a major step toward real automation.

What you are learning today

Lists let you keep related operational values together instead of creating too many separate variables.

create

Create lists

Store multiple values like servers, checks, alerts, environments, file names, or statuses in one variable.

index

Use indexes

Access items using positions. Python starts counting list positions from 0, not 1.

access

Read values

Pick the first server, second check, latest alert, or any specific value from a list.

update

Update values

Change an item inside a list when a status changes from RUNNING to FAILED or PENDING to DONE.

count

Count items

Use len() to count total servers, files, alerts, jobs, or checks for a small report.

automation

Think in groups

Most automation begins when you stop handling one item and start handling a group of related items.
